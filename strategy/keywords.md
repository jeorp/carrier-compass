# キーワード設計

新記事を書く前に必ずここからメインキーワードを選ぶこと。
記事を追加したら `articles.md` と合わせてこのファイルも更新する。

---

## 凡例

- **検索意図**
  - 悩み系：読者が自分の状況を検索する（「営業 辞めたい」「仕事 行きたくない」）
  - ハウツー系：解決策・方法を求めている（「転職前 やること」「副業 始め方」）
  - 比較系：選択肢を比べたい（「大手 スタートアップ 違い」）
  - 体験談系：実例・リアルな声を知りたい（「転職 後悔 体験談」）
- **競合度**：高 / 中 / 低（主観的な定性評価。月次で見直す）

## キーワード設計のルール

1. **メインキーワードは1記事1フレーズ**。複数狙おうとすると共食いになる
2. **競合度が「低」のニッチキーワードを優先**。立ち上げ期は小さく勝てる場所から
3. **タイトル・description・本文冒頭100字**にメインキーワードを自然に含める
4. **サブキーワードは本文中に自然に登場**させる（詰め込まない）
5. **新記事追加時はこのファイルを先に更新**してからスラッグ・タイトルを決める
