## 記事テンプレート

### タイトルの型

- `[悩み] + [解決の示唆]`
- 例：「営業を辞めたいと思ったとき読んでほしい。〇〇した話」
- 例：「転職に踏み切れない人の特徴。〇〇な理由と抜け出し方」

### 本文の構造

```
① 冒頭3行で検索意図に直接応答
   → 「転職したいのに動けない人へ。同じ状況から抜け出した話です」
② 体験談（自分ごと化させる）
③ 気づきや転換点
④ 筆者自身の言葉・独り言のような余韻で締める（読者への問いかけはNG）
⑤ 関連記事2〜3本への自然な誘導
```

### 文字数の目安

1,500〜2,500字。Googleは長さより深さを評価する。

## シリーズ設計

単発記事より、テーマを縦に掘るほうがSEOで強くなる。

### 転職決断シリーズ（syun担当）

1. ✅ 転職を迷い続ける人が陥る思考のクセ
2. ✅ 転職の決断を先延ばしにするとどうなるか
3. ✅ 転職を決断した日、何がきっかけだったか
4. ✅ 転職前に絶対やっておくべきこと
5. ✅ 転職後に後悔しないために確認すること

### 営業キャリアシリーズ（yuki担当）

1. ✅ 仕事が怖くなったとき
2. ✅ 営業を辞めたいと思ったとき
3. ✅ 営業成績が落ちたときの立て直し方
4. ✅ 営業に向いていないと感じたら読む話
5. ✅ 大手営業とスタートアップ営業、何が違うか

### エンジニアキャリアシリーズ（kz担当）

1. ✅ エンジニアの20代キャリア戦略
2. ✅ エンジニアは何を勉強すればいいのか
3. ✅ フリーランスエンジニアになってわかったこと
4. エンジニアが副業を始めるときに知っておくこと（未執筆）
5. AIが普及する時代にエンジニアとして生き残るには（未執筆）

## 内部リンクのルール

- 記事末尾の関連記事は**同シリーズから必ず1本**入れる

- カテゴリページが「シリーズまとめ」の役割を果たすよう記事を揃える
