# マナブスタイル ライティングガイド

マナブ（manablog）の文章構造を参考にした、このブログの品質基準。
syun・kz の記事に適用する。yuki には適用しない（yuki は自然体・感情重視のスタイルを維持）。

---

## 核心：3つの原則

### 1. 結論先出し（BLUF）

H2セクションの冒頭1文で、そのセクションの結論・主張を言い切る。
その後に理由・背景・エピソードを続ける。

**Before（経緯から入る）**
```
## フリーランスを始めた理由

新卒でブロックチェーンゲームのスタートアップに入って、半年でフリーランスに転向した。
理由はシンプルで、責任に報酬が見合っていなかった。
```

**After（結論から入る）**
```
## フリーランスを始めた理由

理由はシンプルで、責任に報酬が見合っていなかった。新卒でブロックチェーンゲームのスタートアップに入って、半年でフリーランスに転向した。
```

---

### 2. 事実・数字優先

感情語より具体的な事実・数字を先に出す。感情は事実の後に添える。

**Before（感情から入る）**
```
インストール数がゼロのまま日が過ぎる感覚が、思った以上にモチベーションにきつく響いた。
```

**After（事実を先に）**
```
インストール数がゼロのまま日が過ぎることは、技術的な詰まりより精神的にきつかった。
```

数字の具体化例：
- 「大きく下がった」→「年収が900万から300万台に落ちた」
- 「長い間迷っていた」→「8ヶ月、転職先の条件を比較し続けた」
- 「時間がかかった」→「慣れるまでに、最初の数ヶ月を消耗した」

---

### 3. 過去の自分を教材にする

「あのころの自分は知らなかった → 今は分かる → だからこう言える」という構造。
読者に「これを読む前の自分」と重ねてもらう。

**使える型**
```
1年目のうちに〇〇しておけばよかった、と今は思っている。（→ 教訓）
あのとき〇〇だったのは、□□を知らなかったからだ。（→ 原因の言語化）
当時の自分に言えるなら、〇〇とだけ伝えたい。（→ 締め）
```

## チェックリスト（H2ごとに確認）

- [ ] 冒頭1文で結論を言い切っているか
- [ ] 感情語より事実・数字が先に来ているか
- [ ] 「良かった点から書く」「次に〇〇について述べる」のようなメタコメントがないか
- [ ] 「これ」「そこ」などの指示語で結論を先送りしていないか（例：「1年目にいちばん効いたのは、これだ。」→ 「これ」が何か言い切る）

## 適用対象外の表現

マナブスタイルは **論理構造** の話であり、文体そのものを変えない。
以下はペルソナの文体として維持する：

- syun の「照れて引く」瞬間（例：「照れくさいけど、それが本当のことだ」）
- kz の「正直まだ分かっていない」という留保
- 体験談の中の感情描写（「怖かった」「しんどかった」）
